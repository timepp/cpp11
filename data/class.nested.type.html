[class.nested.type]9.9 Nested type names

1Type names obey exactly the same scope rules as other names. In particular, type names defined within a class definition cannot be used outside their class without qualification.

[ Example: 

struct X {
  typedef int I;
  class Y { /* … */ };
  I a;
};

I b;                    // error
Y c;                    // error
X::Y d;                 // OK
X::I e;                 // OK

— end example ]
