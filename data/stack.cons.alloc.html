[stack.cons.alloc]23.6.5.4 stack constructors with allocators

1If uses_allocator<container_type, Alloc>::value is false the constructors in this subclause shall not participate in overload resolution.

template <class Alloc> explicit stack(const Alloc& a);

2Effects: Initializes c with a.

template <class Alloc> stack(const container_type& cont, const Alloc& a);

3Effects: Initializes c with cont as the first argument and a as the second argument.

template <class Alloc> <dfn>stack</dfn>(container_type&& cont, const Alloc& a);

4Effects: Initializes c with std::move(cont) as the first argument and a as the second argument.

template <class Alloc> stack(const stack& s, const Alloc& a);

5Effects: Initializes c with s.c as the first argument and a as the second argument.

template <class Alloc> stack(stack&& s, const Alloc& a);

6Effects: Initializes c with std::move(s.c) as the first argument and a as the second argument.
