[temp.decls]14.5 Template declarations

1A template-id, that is, the template-name followed by a template-argument-list shall not be specified in the declaration of a primary template declaration.

[ Example: 

template<class T1, class T2, int I> class A<T1, T2, I> { };     // error
template<class T1, int I> void sort<T1, I>(T1 data[I]);         // error

— end example ]

[ Note: However, this syntax is allowed in class template partial specializations (14.5.5). — end note ]

2For purposes of name lookup and instantiation, default arguments of function templates and default arguments of member functions of class templates are considered definitions; each default argument is a separate definition which is unrelated to the function template definition or to any other default arguments.

3Because an alias-declaration cannot declare a template-id, it is not possible to partially or explicitly specialize an alias template.
