[uncaught]18.8.4 uncaught_exception 

bool uncaught_exception() noexcept;

1Returns: true after the current thread has initialized an exception object (15.1) until a handler for the exception (including unexpected() or terminate()) is activated (15.3). [ Note: This includes stack unwinding (15.2). — end note ]

2Remarks: When uncaught_exception() returns true, throwing an exception can result in a call of terminate() (15.5.1).
